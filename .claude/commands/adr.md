Write a new ADR for: $1

Read `docs/adr/0000-template.md` and every existing ADR first, so the new one is consistent and does not contradict an accepted decision.

Number it sequentially. Include: Status, Context, Decision, Consequences (good and bad), Alternatives considered and why they were rejected.

If this decision supersedes an existing ADR, mark the old one as Superseded and link forward. Never edit an accepted ADR's Decision section.

Remember this feeds the Quantic design and testing document, which is graded on "software and architectural patterns used and reasons used."
